# Contributing to PathFatter

Thanks for your interest! Here's how to contribute:

## Bug Reports

- Open an issue with a clear description and steps to reproduce
- Include macOS version and PathFatter version if possible

## Pull Requests

1. Fork the repo
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Ensure the project builds in Xcode with no warnings
5. Commit with a clear message
6. Open a PR against `main`

## Code Style

- Follow existing Swift conventions in the project
- Keep accessibility support in mind (VoiceOver, Dynamic Type, reduced motion)
- Test your changes with VoiceOver enabled

## Questions

Open an issue and tag it as a question.
